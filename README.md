# SpaceGroupCohomology — a GAP package

Mod-2 cohomology rings and Lieb–Schultz–Mattis anomaly classes for the
230 crystallographic space groups.

Reference: Chunxiao Liu and Weicheng Ye, *Crystallography, group cohomology,
and Lieb–Schultz–Mattis constraints*, SciPost Phys. **18**, 161 (2025)
([arXiv:2410.03607](https://arxiv.org/abs/2410.03607)).

## Installation

Symlink this checkout into a path on your GAP `pkg/` search list, typically
`~/.gap/pkg/SpaceGroupCohomology/`:

```bash
mkdir -p ~/.gap/pkg
ln -s "$(pwd)" ~/.gap/pkg/SpaceGroupCohomology
```

The package depends on [HAP](https://gap-packages.github.io/hap/) (≥ 1.30),
which is loaded automatically, and on [polymake](https://polymake.org)
being installed and on the `PATH` (e.g. `brew install polymake`): the
space-group resolutions are built from HAP's `CrystallographicComplex`,
which shells out to polymake to compute the fundamental-domain vertices
and face lattices.

### Optional: external linear-algebra accelerator

Large computations can offload mod-2 linear algebra to a small bundled C++
tool. Building it is optional — without it the package behaves exactly as
before:

```bash
make -C cpp        # builds bin/sgclinalg (no external dependencies)
make -C cpp test   # self-tests
```

Matrices whose size (rows × columns) exceeds `SGC_LINALG_THRESHOLD`
(default `10^6`) are then handled by `bin/sgclinalg` via a sparse-matrix
file interface; everything smaller stays on GAP's native routines, so
published outputs are unchanged. Set the GAP global
`SGC_LINALG_THRESHOLD := 0;` to force everything through the external tool,
or point the environment variable `SGC_LINALG_PATH` at a binary outside the
package directory.

## Usage

```gap
gap> LoadPackage("SpaceGroupCohomology");
true
gap> SpaceGroupCohomologyRingGapInterface(191);   # any IT number in 1..230
```

That single call prints the mod-2 cohomology ring presentation and the
LSM anomaly classes for the given space group.

## Reading the output

The first block is the cohomology-ring presentation. For example,
`Z2[Ax,Ay,Az]/<R2>` means a polynomial algebra over `GF(2)` on the
listed generators, quotiented by the ideal generated by the printed
relations. Products are written with `.`, powers with `^`, and `+` is
addition mod 2.

The generator names are the package's display names for chosen cohomology
classes. Their first letter indicates the cohomological degree used by the
tables: `A` means degree 1, `B` degree 2, `C` degree 3, `D` degree 4, and
`F` degree 6. Thus `Am.Ba` is a degree-3 monomial and `Dd^2` is a degree-8
monomial.

The lines `R2:`, `R3:`, ... group homogeneous relations by total degree.
Multiple expressions on one `Rk:` line are separate relation generators.
For groups with high-degree generators, additional blocks such as `R8` or
`R12` can appear after the compact header; the detailed `Rk:` lines are the
authoritative relation list.

The final `LSM:` block lists the Lieb-Schultz-Mattis anomaly class for each
irreducible Wyckoff position. The label on the left, such as `4a` or `8b`,
is the Wyckoff position; the expression on the right is the corresponding
degree-3 class in the same ring notation. HAP diagnostic lines beginning
with `#I` are warnings about optional visualization helpers and are not part
of the mathematical output.

## Runtime

All 230 groups completed successfully within 1885 seconds.
Most groups were small: 198 finished in at most 10 seconds, another 27 in
at most 20 seconds, and only 5 took longer than 20 seconds. The slowest
recorded group was IT=227 at 29 seconds. Fresh one-group runs can be slower
than this batch baseline because GAP/HAP startup and system load are visible
at this scale, so treat these numbers as a local reference rather than a
portable guarantee.

## Layout

```
SpaceGroupCohomology/
├── PackageInfo.g       package metadata + HAP/polymake dependencies
├── init.g              forward declarations of data globals
├── read.g              reads data, linalg, homotopy, then functions
├── gap/
│   ├── data.gi         PGGens230, IWP, GENNAMES, funcs230 (230 entries each)
│   ├── linalg.gi       GAP/native and optional C++ mod-2 linear algebra
│   ├── homotopy.gi     resolution construction with contracting homotopies
│   └── functions.gi    library of cohomology-ring routines
├── cpp/                optional sgclinalg accelerator
└── tst/
    ├── smoke.tst       package self-test
    ├── homotopy.tst    resolution/homotopy sanity checks
    └── linalg.tst      linear-algebra dispatch correctness checks
```

## Testing

`tst/smoke.tst` does two things in order:

1. A constant-time sanity check that all four data tables have 230 entries.
2. Five real cohomology computations — space groups No. 1, 16, 22, 108,
   and 219 — that exercise the full HAP-backed pipeline end-to-end and pin
   down the expected output. Any regression in the cup product, relations,
   or LSM-class code will then surface as a `Test` failure instead of
   passing silently.

```gap
gap> TestPackage("SpaceGroupCohomology");
```

The first item is instantaneous; the pinned computations take about a minute
on a modern laptop, with group 219 the slowest of the smoke-test groups.
